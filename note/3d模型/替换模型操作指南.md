# 替换模型操作指南

本文档旨在指导你如何修改代码，实现真正的"换装"功能（替换模型），而不仅仅是"换色"（替换材质）。

## 核心概念解释（写给零基础的你）

在开始修改代码之前，我们需要先理解两个概念，这样你知道我们在改什么。

### 1. 什么是"骨架网格体 (Skeletal Mesh)"？
想象它就是角色的"身体部件"。
- 之前我们修改材质，相当于给角色**涂油漆**，形状没变，只是颜色变了。
- 现在我们要替换模型，相当于给角色**换衣服**，比如把短袖换成铠甲，形状会发生改变。

### 2. 什么是"Leader Pose (主姿态)"？
这是模块化角色（Modular Character）最关键的技术。
- **问题**：如果你的角色由头、身体、手、脚四个部分拼接而成，当角色跑步时，如果这四个部分各自跑各自的，可能会出现"手跟不上身体"或者"头身分离"的恐怖画面。
- **解决**：我们需要指定一个"老大"（Master，通常是身体或核心骨架），然后告诉其他部位（Slave，如手套、靴子）：**"老大怎么动，你就怎么动"**。
- **术语**：在代码中，这叫做 `SetLeaderPoseComponent`（设置主姿态组件）。只要设置了这个，手套就不需要自己播放动画，它会完美吸附在身体的骨骼上运动。

## 代码修改步骤

请打开文件 `G:\ueprj\ue_demo_worldgame\Source\worldgame\Private\Yy\Character\AvatarTestComponent.cpp`，按照以下步骤修改。

### 第一步：修改初始化逻辑 (`SetupSlave`)

我们需要在组件初始化时，建立"老大"（Master）和"小弟"（Slave）的跟随关系。

**找到代码位置**：约第 109 行 `SetupSlave` 函数。

**修改建议**：
1. **删除/注释** `SetAnimClass`：衣服不需要有自己的"大脑"（动画蓝图），它只需要跟随身体。如果给衣服也设置动画蓝图，不仅浪费性能，还容易导致动作不同步。
2. **启用** `SetLeaderPoseComponent`：建立跟随关系。

**修改后的代码**：

```cpp
void UAvatarTestComponent::SetupSlave(FName SlotName, USkeletalMeshComponent* Slave, const TArray<FName>& BonesToHide)
{
    if (!Slave || !MasterMesh) return;
    UE_LOG(LogTemp, Warning, TEXT("[AvatarTest] SetupSlave: Slot='%s', HideBones=%d"), 
        *SlotName.ToString(), BonesToHide.Num());
    
    // [修改点 1] 启用 Leader Pose，让部件跟随主网格运动
    // 这行代码的作用：告诉 Slave (比如手套)，你的骨骼位置完全听 MasterMesh (身体) 的指挥。
    Slave->SetLeaderPoseComponent(MasterMesh);

    // [修改点 2] 注释掉 SetAnimClass
    // 既然已经跟随了主姿态，Slave 就不需要自己播放动画了。
    // Slave->SetAnimClass(MasterMesh->GetAnimClass());

    // ... 下面的隐藏骨骼逻辑保持不变 ...
    for (const FName& BoneName : BonesToHide)
    {
        Slave->HideBoneByName(BoneName, PBO_None);
        UE_LOG(LogTemp, Warning, TEXT("[AvatarTest]   HideBone: '%s'"), *BoneName.ToString());
    }
    // ...
}
```

### 第二步：修改换装逻辑 (`SwapSlotMesh`)

当你需要换衣服时，我们需要确保新换上的衣服也能正确跟随老大。

**找到代码位置**：约第 156 行 `SwapSlotMesh` 函数。

**修改建议**：
1. **取消注释** `SetLeaderPoseComponent`：虽然初始化时设置过，但为了保险起见（防止换了模型后关系丢失），建议再次确认跟随关系。

**修改后的代码**：

```cpp
void UAvatarTestComponent::SwapSlotMesh(FName SlotName, USkeletalMesh* NewMesh)
{
    // ... 前面的查找逻辑不变 ...
    USkeletalMeshComponent** SlavePtr = SlaveMap.Find(SlotName);
    if (!SlavePtr || !(*SlavePtr))
    {
        // ... 日志 ...
        return;
    }
    // ... 空指针检查 ...

    USkeletalMeshComponent* SlaveComp = *SlavePtr;

    // [修改点 3] 替换模型资产
    // 这行代码的作用：把当前的 3D 模型（比如旧外套）换成新的（NewMesh，比如新铠甲）
    SlaveComp->SetSkeletalMeshAsset(NewMesh);

    // [修改点 4] 再次确保跟随关系
    // 换了新模型后，告诉它："你还是要跟着老大动哦"
    SlaveComp->SetLeaderPoseComponent(MasterMesh);

    // ... 下面的隐藏骨骼逻辑保持不变 ...
}
```

## 如何使用（在编辑器中）

修改完 C++ 代码并**编译**成功后，回到虚幻引擎编辑器。

1. **准备资源**：确保你已经导入了拆分好的骨骼模型（比如单独的手套模型、靴子模型），并且它们是**使用同一套骨骼（Skeleton）**导入的。如果骨骼不匹配，Leader Pose 会失效（衣服会变成 T-Pose 或乱飞）。
2. **调用函数**：
   - 在你的角色蓝图或测试逻辑中，不要再调用 `SwapSlotMaterial`。
   - 改为调用 `SwapSlotMesh`。
   - **参数说明**：
     - `SlotName`: 填入你要换的部位名称（如 "Hands", "UpperBody"）。这些名字要和你 `BeginPlay` 里定义的对应。
     - `NewMesh`: 选择你要换上的那个骨骼网格体资源。

## 常见问题排查

- **Q: 换装后衣服不跟着人动？**
  - A: 检查 `SetupSlave` 和 `SwapSlotMesh` 里是否都执行了 `SetLeaderPoseComponent`。
  - A: 检查你的衣服模型和身体模型是否**共用同一个 Skeleton 资产**。

- **Q: 换装后衣服穿模（肉露出来了）？**
  - A: 这通常是美术模型的问题。你需要检查 `SlotHiddenBones` 配置，确保穿上厚衣服时，里面的身体骨骼被隐藏了（代码里已经有隐藏骨骼的逻辑了，你可能需要调整隐藏列表）。

- **Q: 之前的换颜色功能还能用吗？**
  - A: 能用。我们保留了 `SwapSlotMaterial` 函数，你依然可以用它来改变衣服的颜色（材质）。
